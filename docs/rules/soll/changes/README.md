# Change Documents (Soll-Regeln)

Änderungen am Soll-Regelsystem (`docs/rules/soll/aufbausystem.md`) werden als **nummerierte Change Documents** festgehalten.

Prinzip:
- **Docs zuerst**: Eine Regeländerung gilt erst dann als beschlossen, wenn sie als Change Document vorliegt und in `docs/rules/soll/aufbausystem.md` eingepflegt ist.
- **Implementierung danach**: Engine/LLM/Playtests werden anschließend angepasst. Abweichungen/Status stehen in `docs/rules/rules-v1.md` (implementiert) und `docs/dev/implementation-status.md`.
